# Node.js (JS/TS): 잡 크롤러 및 자동화 시스템에서의 장점

## 1. 프론트엔드와의 언어 통일 (Full-Stack)
- 사용자가 크롤링된 결과를 보고, 이력서가 자동 생성되는 과정을 제어할 별도의 웹 서비스(React, Vue 등)를 구축한다면 백엔드부터 프론트엔드까지 모두 JavaScript/TypeScript로 통일할 수 있습니다.
- 풀스택 개발자로서의 강점을 보여주는 포트폴리오를 계획하신다면 매우 유리한 선택지입니다.

## 2. 강력한 비동기(Asynchronous) I/O 모델
- Node.js의 핵심 특징인 논블로킹 비동기 I/O는 여러 취업 사이트로 동시에 HTTP 요청을 보내고 응답을 기다리는 '크롤러'의 특성에 아주 잘 맞습니다. 
- 적은 메모리 자원으로도 빠르고 효율적으로 여러 개의 스크래퍼를 병렬 실행하기 좋습니다.

## 3. 최신 헤드리스 브라우저와의 찰떡 궁합
- 동적 사이트(React 등으로 만들어진 페이지)를 크롤링할 때 필수적인 헤드리스 브라우저 자동화 도구인 `Puppeteer`나 `Playwright`는 모두 Node.js 기반에서 탄생하고 가장 네이티브하게 잘 동작합니다.
- 브라우저 API를 다루는 방식이라 JavaScript 개발자에게 가장 직관적입니다.
