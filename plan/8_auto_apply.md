## 8.2 로그인 자동화 (Session Management)

### 외부 계정 관리
```java
@Entity
public class ExternalAccount {
    @Id @GeneratedValue
    private Long id;
    
    @ManyToOne
    private User user;
    
    @Enumerated(EnumType.STRING)
    private SourceSite site;           // SARAMIN, JOBPLANET
    
    private String loginId;            // 사이트 로그인 ID
    
    @Column(length = 500)
    private String encryptedPassword;  // AES-256 암호화된 비밀번호
    
    private String sessionData;        // Redis key 참조
    private LocalDateTime lastLoginAt;
    private boolean isValid;           // 세션 유효 여부
}
```

### 세션 관리 플로우
```
1. 사용자가 사람인/잡플래닛 계정 정보를 등록 (AES-256 암호화 저장)
2. Auto-Apply 최초 실행 시 Playwright로 대리 로그인
3. 로그인 성공 → 쿠키/세션 정보를 Redis에 저장 (키: session:{userId}:{site})
4. 이후 지원 시 Redis에서 세션 로드 → Playwright 컨텍스트에 주입
5. 세션 만료 감지 시 자동 재로그인
6. **각 사용자별 독립적인 세션** 관리
```

### 보안 고려사항
- 비밀번호는 반드시 **AES-256 암호화** 후 DB 저장
- 복호화 키는 환경 변수로 분리 (코드에 하드코딩 금지)
- Redis 세션 데이터 TTL: 24시간 (이후 자동 재로그인)
- HTTPS 전용 통신


### [프로젝트 다시 선택] 기능
```
1. 사용자가 [프로젝트 다시 선택] 버튼 클릭
2. 등록된 내 프로젝트 목록이 모달로 표시
3. 사용자가 원하는 프로젝트를 1~3개 선택
4. AI가 새로 선택된 프로젝트 기반으로 자소서/포폴 텍스트 재생성
5. 재생성된 텍스트가 오른쪽 패널에 업데이트
```

### 진입 경로
- **디스코드 알림 링크 클릭** → 해당 공고의 Preview 페이지로 바로 이동
- **공고 카드의 "지원하기" 버튼** → Preview 페이지로 이동
- 진입 시 **AI 자소서/포폴이 이미 사전 생성**되어 있음 (즉시 검토 가능)

## 8.4 자동 지원 실행 (Playwright 로봇)

### 즉시지원 (DIRECT_APPLY) 플로우
```
1. 사용자가 Preview 페이지에서 [최종 지원] 버튼 클릭
2. 백엔드에서 Playwright 로봇 가동
3. Redis에서 해당 사용자의 사이트 세션 로드
4. 세션 주입 → 공고 페이지 접근
5. "입사지원" 버튼 클릭
6. 지원 폼에 데이터 입력:
   - 자소서 텍스트 → 해당 입력 필드에 주입
   - 포트폴리오 PDF → setInputFiles()로 첨부
   - 추가 파일(증명서 등) → setInputFiles()로 첨부
7. "제출" 버튼 클릭
8. 결과 확인 (즉시 검증)
```

### 홈페이지지원 (HOMEPAGE) 처리
```
1. [최종 지원] 버튼 클릭
2. 해당 회사 채용 홈페이지 URL을 새 탭/창으로 열기
3. 사용자가 직접 지원 (자동화 불가)
4. 지원 후 웹앱에서 "지원 완료" 수동 버튼 클릭
```

## 8.5 지원 성공/실패 2단계 검증

### 1단계: 즉시 확인
```
지원 제출 직후:
├── 성공 메시지/페이지 DOM 감지 
│   └── → status = APPLIED (지원 완료)
├── 에러 메시지 감지 
│   └── → status = FAILED + failReason 저장
└── 타임아웃 (30초)
    └── → status = UNKNOWN + 사후검증 대기
```

### 2단계: 사후 교차검증
```
스케줄링 (매 6시간):
1. 사람인/잡플래닛의 "내 지원 현황" 페이지 크롤링
2. 실제 지원 목록과 우리 DB의 APPLIED 항목 대조
3. DB에는 APPLIED인데 실제로 없는 경우 → FAILED로 업데이트 + 알림
4. 성공 확인된 항목 → VERIFIED로 업데이트
```

### 실패 시 처리
```
1. 디스코드 알림 발송: "❌ [회사명] 지원 실패 - 사유: {failReason}"
2. 웹앱 내 에러 알림 배지 표시
3. 실패 상세 페이지에서:
   - 실패 사유 표시
   - [재시도] 버튼 제공
   - [수동 지원] 링크 (홈페이지로 이동)
```
