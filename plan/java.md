# Java (Spring Boot): 잡 크롤러 및 자동화 시스템에서의 장점

## 1. 기존 경험과의 시너지 및 취업 시장 적합성
- 이전에 진행하셨던 기존 프로젝트(오프서비스 등)가 이미 Spring Boot 기반입니다. 해당 경험과 연계하여 백엔드 역량을 심도 있게 파고들 수 있습니다.
- 국내 백엔드 개발자 취업 시장에서 가장 수요가 많은 언어이므로, 크롤러를 Spring Boot로 안정성 있게 구현하는 것 자체가 강력한 취업용 무기가 됩니다.

## 2. 대규모/배치 처리에서의 구조적 안정성
- 엔터프라이즈 환경에서 데이터를 다루는 방법을 포트폴리오에 녹일 수 있습니다.
- 예를 들어 **Spring Batch**와 연동하여 '매일 자정에 사람인/원티드 채용 공고를 긁어오고 파싱하여 DB에 밀어넣는 자동화 잡(Job)'을 구현하면 백엔드 아키텍처로써 매우 훌륭한 그림이 나옵니다.
- 객체 지향적인 설계와 강력한 타입 시스템으로 나중에 규모가 커져도 유지보수가 용이합니다.

## 3. Java 생태계의 스크래핑 도구
- Jsoup(정적 크롤링), Selenium Java 바인딩 등 크롤링을 위한 안정적인 라이브러리를 사용할 수 있어 기본적인 수집 역량도 충분합니다.
