# 취업 사이트 크롤러 및 이력서/포트폴리오 자동화 프로젝트 플랜

## 1. 프로젝트 개요
- **목표**: 주요 취업 포털(사람인, 잡플래닛 등)의 채용 공고를 수집하고, AI 기반 맞춤형 자소서/포트폴리오를 자동 생성하는 **다중 사용자 지원** 서비스 구축
- **사용자 인증**: 각 사용자가 회원가입/로그인(JWT, **토큰 만료 24시간**) 후 자신만의 프로필, 프로젝트, 템플릿, 외부 사이트 계정 정보를 독립적으로 관리
- **외부 사이트 로그인**: 각 사용자가 사람인/잡플래닛 계정을 **개별적으로 등록**하여 자신의 계정으로 입사 지원 가능
- **주요 기능**:
  1. **잡 크롤러 (Job Crawler)**: 맞춤형 채용 공고 수집 및 키워드 파싱 (기술 스택, 우대 사항 등)
  2. **데이터 가공 (Data Processing)**: 수집된 데이터 정규화, DB 적재 및 분석
  3. **자동화 (Resume/Portfolio Automation)**: 공고별 맞춤형 이력서/커버레터/포트폴리오 초안 자동 생성 (OpenClaw HTTP API 연동)
  4. **자동 지원 (Auto-Apply) 및 권한 연동 기능**
  5. **디스코드(Discord) 알림 연동**
  6. **프론트엔드 연동 및 최종 서비스화 (반응형 웹앱)**
  7. **배포 파이프라인 (CI/CD) 구축**

## 2. 단계별 진행 계획
- **Step 1: 요구사항 정의 및 기술 스택 선정**
  - 타겟 취업 사이트: **사람인(Saramin), 잡플래닛(JobPlanet)**
  - 핵심 언어 및 프레임워크: **Java (Spring Boot) + Playwright Java**
- **Step 2: 핵심 크롤러 엔진 개발**
  - 사이트별 크롤링 모듈 구현 (Playwright 기반 동적 렌더링 대응)
  - **차단 방지(Anti-Bot Bypass) 고도화**: 사람인, 잡플래닛 등 강력한 크롤러 차단 봇(Incapsula, Cloudflare 등) 우회를 위한 전략 수립 (User-Agent 동적 변경, 사람 손과 유사한 랜덤 딜레이 구현, Playwright Stealth 플러그인 연동 및 Headless 우회 등)
- **Step 3: 데이터 파이프라인 및 저장소 구축**
  - **크롤링된 데이터 정제 및 메타 정보 수집**: `JobPosting` DB 보관 시, 제목/회사명 외에도 학력, 경력, 연봉 정보와 더불어 **"입사 지원 방식(예: 사람인 즉시지원, 홈페이지 지원, 이메일 지원 등)"의 메타 데이터를 필수적으로 정제 및 파싱**하여 저장
  - **AI 적합률 분석 + 프로젝트 자동 매칭**: 공고가 크롤링되면 AI가 내 프로필 기반으로 적합률을 산출하고, **해당 공고가 요구하는 스킬에 맞는 내 프로젝트를 자동으로 선별·매칭**하여 자소서/포폴을 바로 생성 (팝업 없이 즉시 분석)
  - **초기 크롤링 버튼 (일회용)**: 처음 사용 시 누르면 **마감되지 않은 활성 공고만 전부 크롤링**해오는 버튼. 크롤링 차단 봇 고려하여 램덤 딜레이/UA 변경 등 Anti-Bot 전략 적용
  - **스케줄링(Scheduling)**: 이후부터는 Spring `@Scheduled`나 Quartz를 이용하여 자동 수집 및 업데이트
- **Step 4: 내 프로필 및 프로젝트 관리 페이지**
  - **기본 프로필**: 이력서(학력, 경력, 자격증)/ 기술 스택 / 강점 등 입력·수정·관리 (이력서는 기본 이력서로 사용자 정의 가능)
  - **희망 직무 설정**: **사이트별로 직무 카테고리가 다르므로, 각 사이트의 실제 분류 체계를 그대로 가져와서 사이트별로 복수 선택(멀티)** 가능. 해당 직무 공고만 디스코드 알림 + AI 사전 생성. **완전히 끄는 OFF 토글**도 제공
  - **내 프로젝트 관리 (위시켓 스타일)**: 프로젝트별로 GitHub 주소, 노션 링크, 설명 글 작성, 이미지 첨부 등을 등록하는 전용 페이지
  - **AI 프로젝트 정리 탭**: GitHub 주소를 입력하면 AI가 레포 내 코드 및 `.md` 파일을 읽어서 프로젝트를 자동 정리. **정리된 결과에 템플릿 적용 및 이미지 삽입이 가능**하고, 완성되면 **새로운 프로젝트 항목으로 저장**되어 향후 AI 자소서/포폴 생성 시 읽을 수 있는 데이터로 활용
  - 이 데이터가 AI 자소서/포트폴리오 생성과 적합률 분석의 기반 소스로 활용됨
- **Step 5: 자소서/포트폴리오 자동화 및 멀티 템플릿 시스템**
  - **이력서(Resume) 고정**: 사용자의 기본 이력서(학력, 기본 경력 등)는 한 번 저장하면 모든 지원에 **공통으로 재사용**
  - **사전 기업/직무 분석 (Pre-analysis)**: 공고 클릭 시 OpenClaw HTTP API가 해당 기업/직무 분석 인사이트 제공
  - **AI 자소서/포트폴리오 자동 작성**: 분석된 기업 정보와 내 고정 이력서/경험을 바탕으로, **AI는 오직 맞춤형 자기소개서와 포트폴리오 텍스트만 생성**
  - **멀티 템플릿(Multi-Template) 시스템**: 사용자가 여러 개의 템플릿을 미리 저장해 두고, 공고 성격에 맞춰 원하는 템플릿을 선택하여 텍스트를 주입. **(단, 생성 결과물과 템플릿에 마크다운 문법(`*`, `#` 등)은 절대 사용 불가. 순수 텍스트/HTML/Plain 폼만 허용)**
- **Step 6: 프론트엔드 연동 및 최종 서비스화 (반응형 웹앱)**
  - **상단 네비게이션 바**: "이끼잡" 로고/텍스트 배치, 클릭 시 홈 페이지로 이동
  - **메인 페이지 레이아웃**: 기본은 **카드형 공고 목록** (모바일 1열, 데스크탑 3열 그리드) + **리스트(테이블)형 토글 전환** 버튼 제공
  - **사이트별 탭 분리**: 사람인 탭 / 잡플래닛 탭으로 공고 분리 표시. **각 사이트마다 데이터 구조/템플릿이 다르므로, 탭별로 해당 사이트에 맞는 카드/상세 UI 템플릿으로 렌더링**. 검색은 전체 탭 공통이되, 결과는 현재 탭의 공고만 표시
  - **지원 상태 표시**: 즉시지원 공고는 자동으로 "지원 완료" 라벨 부여. **홈페이지 지원 공고는 자동 확인이 불가하므로, 사용자가 직접 "지원 완료" 버튼을 눌러 수동 표시** 가능
  - REST API 제공 및 사용자 입력 UI 개발
  - **고급 필터링/정렬**: 실제 사람인/잡플래닛 사이트의 필터 항목을 그대로 벤치마킹하여 구현 (직종, 개발 분야, 학력, 경력, 연봉, 근무지역 등)
  - 데스크탑(웹)과 모바일 모두 앱처럼 동작하는 **반응형 웹앱(PWA 고려)** 형태로 구현(기본 메인은 웹)
- **Step 7: 배포 파이프라인 (CI/CD) 구축**
  - **테스트 환경**: 로컬 서버 + **Cloudflare Tunnel** → 도메인: **job.eekky.com** (완전 무료)
  - **프로덕션 환경**: 추후 결정 (Oracle Cloud Free Tier / AWS Lightsail 등)
  - **GitHub Actions 연동**: `git push` 시 자동 빌드 및 배포
  - **서버 배포 스크립트 (PM2 적용)**: `git pull` → `./gradlew build` → **PM2**로 Spring Boot 재구동 및 무중단 운영
- **Step 8: [고도화 목표] 자동 지원 (Auto-Apply) 및 권한 연동 기능**
  - **로그인 자동화 (Session Management)**: **각 사용자별로** 사람인/잡플래닛 계정 정보를 암호화해 등록하고, Playwright 대리 로그인으로 발급된 세션을 해당 사용자 전용 Redis/DB에 유지
  - **입사 지원 전 사전 검토 페이지 (Preview/Edit/Attach)** — 2분할 레이아웃:
    - **왼쪽**: 해당 회사의 기업 데이터 + 공고 상세 정보 + 회사가 공고에 올린 이미지
    - **오른쪽**: 내 이력서 + AI가 자동 선별한 프로젝트 기반으로 생성된 자소서/포트폴리오 (수정/템플릿 변경 가능) + 추가 파일 첨부 + 입사 지원 버튼
    - **[프로젝트 다시 선택] 버튼**: AI가 자동 선별한 프로젝트가 마음에 안 들면 사용자가 직접 프로젝트를 다시 선택 → AI가 글을 새로 생성
  - **다중 파일 첨부 및 폼 로봇 (Submit)**: 검토가 끝나고 "최종 지원" 버튼을 누르면, Playwright 로봇이 생성된 포트폴리오 PDF와 사용자가 올린 기타 증명서 파일들을 `setInputFiles()`로 한꺼번에 폼에 첨부하고 지원을 완료합니다. (단, 홈페이지 지원 공고일 경우 새 창 띄우기로 전환)
  - **지원 성공/실패 확인 (2단계 검증)**:
    1. **즉시 확인**: 제출 후 성공 메시지/페이지 DOM 감지 → 성공이면 "지원 완료", 에러면 실패 알림
    2. **사후 교차검증**: 사람인/잡플래닛의 "내 지원 현황" 페이지를 주기적으로 크롤링해서 실제 지원 목록과 대조
  - 지원 실패 시 디스코드 + 웹앱 내 에러 알림 발송 및 실패 사유와 재시도(Retry) 버튼 제공
- **Step 9: 디스코드(Discord) 알림 연동**
  - **사용자별 Webhook URL 등록**: 각 사용자가 프로필 설정에서 자신의 디스코드 Webhook URL을 등록
  - **희망 직무 기반 필터링**: 사용자가 설정한 희망 직무에 매칭되는 새 공고만 알림 발송 (불필요한 공고 알림 방지)
  - **딥링크(원클릭 지원 진입)**: 알림 링크 클릭 → 웹앱 Preview/Edit 페이지로 바로 이동 (이미 AI가 자소서/포폴을 만들어 놓은 상태)
- **Step 10: AI 테스트**
  - **AI 테스트**: 기능 구현 후 체크 박스 형태로 체크 하듯이 기능 구현 여부 체크 

## 3. 기술 스택 확정 내역
현재까지 논의되어 확정된 메인 아키텍처는 다음과 같습니다.
- **Backend Framework**: Java (Spring Boot v3+) / **Domain-Driven Design (DDD)** 아키텍처 (Service는 흐름만 제어, 핵심 비즈니스 로직은 Domain 객체 위임)
- **Crawler Engine**: Playwright Java (헤드리스 크로미움 제어 및 동적 DOM 파싱)
- **Database**: PostgreSQL via Spring Data JPA
- **Caching**: Redis (세션 캐싱, 크롤링된 중복 공고 체크, API 응답 속도 최적화 등)
- **Frontend**: React + **shadcn/ui** (Tailwind CSS 기반 컴포넌트 라이브러리)
- **AI Automation**: OpenClaw HTTP API 를 활용한 자소서/포트폴리오 맞춤형 텍스트 생성
- **Template System**: 다중 저장 및 선택 가능한 사용자 맞춤형 템플릿 시스템 (**마크다운 절대 금지**, Plain Text/양식 기반)
- **Deployment**: 테스트는 **로컬 서버 + Cloudflare Tunnel** (무료), 프로덕션은 추후 결정. GitHub Actions CI/CD + PM2
