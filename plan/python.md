# Python: 잡 크롤러 및 자동화 시스템에서의 장점

## 1. 압도적인 크롤링 및 스크래핑 생태계
- **라이브러리의 성숙도**: `BeautifulSoup`, `requests`와 같은 쉽고 강력한 정적 크롤링 도구부터 `Scrapy` 같은 엔터프라이즈급 크롤링 프레임워크까지 완벽하게 지원합니다.
- **동적 페이지 처리**: `Selenium` 및 `Playwright` 파이썬 바인딩 등 동적으로 로드되는 모던 웹사이트를 크롤링할 수 있는 정보와 커뮤니티 예제가 가장 많습니다.

## 2. 데이터 처리 및 인공지능(LLM) 연동의 강력함
- 이력서/포트폴리오 자동화 단계에서 수집된 채용정보 키워드를 뽑고, 언어모델(OpneAI API, LangChain 등)을 이용해 텍스트를 요약하고 생성하는 로직을 붙인다면 파이썬보다 좋은 선택지는 없습니다.
- 크롤링한 방대한 양의 데이터를 가공하는데 `Pandas` 등의 강력한 도구들을 사용할 수 있어 전처리 과정이 수월해집니다.

## 3. 빠른 개발 속도 (Prototyping)
- 문법이 간결하여 아이디어를 코드로 옮겨 동작을 확인하는 속도가 매우 빠릅니다. 처음부터 대규모 시스템을 만들기 보단 빠르게 핵심 가치(크롤링 + 자동화)를 검증하고 고도화하는데 유리합니다.
