# AI 프롬프트 가이드

OpenClawAiGenerator에서 사용하는 모든 AI 프롬프트 정리.
파일: `infrastructure/ai/OpenClawAiGenerator.java`

---

## 프롬프트 목록 (8종)

| # | GenerationType | 메서드 | 용도 | 출력 형식 |
|---|---------------|--------|------|----------|
| 1 | `COVER_LETTER` | `buildCoverLetterPrompt()` | 기본 자소서 생성 | 순수 텍스트 |
| 2 | `CUSTOM_COVER_LETTER` | `buildCustomCoverLetterPrompt()` | 커스텀 자소서 (문항별) | JSON 배열 |
| 3 | `PORTFOLIO` | `buildPortfolioPrompt()` | 기본 포트폴리오 생성 | 순수 텍스트 |
| 4 | `CUSTOM_PORTFOLIO` | `buildCustomPortfolioPrompt()` | 커스텀 포트폴리오 (문항별) | JSON 배열 |
| 5 | `COMPANY_ANALYSIS` | inline (switch case) | 기업 분석 | 마크다운 |

## 1. COVER_LETTER (기본 자소서)

**역할**: 한국어 IT 업계 자소서 컨설턴트

**입력 데이터**:
- 사용자 프로필 (학력, 경력, 기술스택)
- 채용 공고 상세
- 기업 분석 결과 (캐시)
- 매칭 프로젝트 상세 (최대 3개)
- 출처 사이트 (사이트별 가이드 분기)

**사이트별 가이드**:
- SARAMIN/JOBKOREA: 합격자소서 5건 분석 패턴 반영 (소제목, 정량성과, 솔직한계)
- LINKAREER: 대학생/신입 타겟, 학습 과정과 성장 강조
- JOBPLANET: 실무 경험 기반 문제 해결 역량 강조

**핵심 규칙**:
- 존댓말 통일 (~합니다/~했습니다)
- 추상적 표현 금지 ("열정", "열심히")
- 에피소드: 상황→행동→결과→교훈 (STAR)
- 정량적 성과 필수
- 시행착오, 트레이드오프 솔직하게 포함
- 후속 제안 문구 절대 금지

**출력**: 순수 텍스트 (5000자 기준)

## 2. CUSTOM_COVER_LETTER (커스텀 자소서)

**역할**: 한국 IT 업계 자소서 컨설턴트

**입력 데이터**:
- 기본 자소서와 동일 + customSections JSON + additionalRequest

**핵심 규칙** (기본 자소서 + 추가):
- 문항별 JSON의 title과 rule을 읽고 해당 규칙에 맞춰 작성
- 각 문항의 내용이 서로 겹치지 않도록
- **프로젝트 분배 규칙**:
  - 프로젝트 여러 개 → 문항별 가장 적합한 프로젝트 배정, 반복 금지
  - 지원동기 → 회사 사업 접점 프로젝트
  - 직무역량 → 공고 기술스택 매칭 프로젝트
  - 문제해결 → 트러블슈팅/성능개선 프로젝트
  - 성장경험 → 새 기술 학습/시행착오 프로젝트
  - 프로젝트 1개 → 문항마다 다른 측면 강조

**출력**: JSON 배열
```json
[{"title":"문항제목1","content":"작성된 내용1"}, ...]
```

## 3. PORTFOLIO (기본 포트폴리오)

**역할**: 한국 IT 업계 채용 전문가

**입력 데이터**:
- 사용자 프로필
- 채용 공고
- 매칭 프로젝트 상세
- 출처 사이트

**포트폴리오 6개 섹션**:
1. 프로젝트 개요 - 한 줄 요약
2. 기술적 의사결정 - 왜 이 기술/아키텍처를 선택했는지 (수치 없이)
3. 핵심 구현 내용 - 직접 구현한 기능 목록 3~5문장
4. 문제 해결 과정 - 디버깅, 시행착오, 트레이드오프 솔직하게
5. 성과 및 수치 - 불릿 한 줄씩, 빈 줄 없이
6. 직무 연관 역량 - 공고 요구사항과 연결 2~3문장

**핵심 규칙**:
- 섹션 간 내용 절대 겹치지 않도록
- 건조한 "~적용 → ~달성" 패턴 금지
- 구어체와 체감 표현 사용
- 문제 해결: 실제 겪은 사람이 쓴 것처럼

**출력**: 순수 텍스트

---

## 4. CUSTOM_PORTFOLIO (커스텀 포트폴리오)

**역할**: 한국 IT 업계 채용 전문가

**입력 데이터**:
- 커스텀 자소서와 동일 구조 (customSections + additionalRequest)

**프로젝트 분배 규칙**:
- 프로젝트 개요 → 가장 대표적인 프로젝트
- 기술적 의사결정 → 아키텍처 선택이 돋보이는 프로젝트
- 문제 해결 → 트러블슈팅/성능개선 프로젝트
- 성과/수치 → 정량적 결과가 명확한 프로젝트
- 프로젝트 1개 → 문항마다 다른 측면 강조

**출력**: JSON 배열 (커스텀 자소서와 동일 형식)

## 5. COMPANY_ANALYSIS (기업 분석)

**역할**: 기업 리서치 전문가

**입력 데이터**:
- 기업 정보 (웹 검색 결과 포함)
- 채용 공고 상세

**분석 항목**:
1. 기업 개요 (업종, 규모, 매출, 설립연도)
2. 비전과 핵심 가치
3. 주요 기술/사업 영역 (경쟁사 대비 차별점)
4. 기업 문화 및 근무 환경
5. 포트폴리오/자소서 작성 시 참고 포인트

**출력**: 마크다운 (유일하게 마크다운 허용)

## 6. PROJECT_SUMMARY (프로젝트 AI 분석)

**역할**: 대기업 채용 면접관 출신 10년차 시니어 개발자

**입력**: GitHub 레포지토리 데이터 (소스코드, README, docs/, 빌드파일)

**출력 JSON**:
```json
{
  "projectName": "실제 서비스명",
  "description": "STAR 구조 설명 (한줄요약/배경/목표/구현/성과)",
  "techStack": "버전 포함 기술 스택",
  "keyFeatures": ["기능명: WHAT + HOW + WHY", ...],
  "architecture": "카테고리별 분리 (계층구조/디자인패턴/데이터저장/성능/보안)",
  "challenges": ["항목명(수치): 문제→원인→해결→결과", ...]
}
```

**핵심 규칙**:
- docs/ 기술 블로그에서 정량적 수치 필수 추출
- description 구현 항목: `항목명(수치): 문제→원인→해결→결과` 형식
- challenges: 5~8개, 수치 없는 항목 제외
- 클래스명 직접 노출 금지 (기술적 역할로 설명)

## 프롬프트 데이터 빌더 (AiPromptDataBuilder)

프롬프트에 전달되는 데이터를 구성하는 헬퍼 클래스.

| 메서드 | 설명 |
|--------|------|
| `buildProfileString(profile, site)` | 사용자 프로필 텍스트 (사이트별 전략 패턴) |
| `buildJobString(job)` | 공고 기본 정보 (제목, 회사, 위치, 기술스택 등) |
| `buildDetailedJobString(job)` | 공고 상세 (기업 분석용) |
| `buildJobStringWithOcr(job)` | 공고 + OCR 이미지 텍스트 (적합률 계산용) |
| `buildMatchedProjectsDetail()` | 매칭 프로젝트 상세 (이름, 설명, 기술스택, GitHub, AI요약) |

### 사이트별 프로필 전략 (SiteProfileStrategy)
- `SaraminProfileStrategy`: 사람인 맞춤 프로필 강조점
- `JobKoreaProfileStrategy`: 잡코리아 맞춤
- `JobPlanetProfileStrategy`: 잡플래닛 맞춤
- `LinkareerProfileStrategy`: 링커리어 맞춤 (대학생/신입 타겟)

## 공통 규칙 (모든 프롬프트)

1. **어투**: 존댓말 (~합니다/~했습니다)
2. **마크다운**: 기업 분석만 허용, 나머지 전부 순수 텍스트
3. **후속 제안**: "다음 단계", "원하시면" 같은 문구 절대 금지
4. **추상 표현**: "열정", "열심히", "확장성 확보" 같은 표현 금지
5. **정량적 수치**: 가능한 모든 곳에 숫자, 전후 비교
6. **프로젝트 분배**: 여러 개면 문항별 적합한 프로젝트 배정, 1개면 다른 측면 강조
